# 天外家园 · 工作分配与界面重构设计草案 v4

> 本文档**只做设计，暂不修改代码**。包含三部分：（1）角色工作分配系统；（2）把侧边栏参数设定移入游戏初始化、游戏中锁定；（3）新增角色界面。

---

## 一、角色工作分配系统（Crew Jobs）

### 核心规则

- **每个角色每天只能分配一项工作**（逐人分配，呼应已有的逐人心情系统）。
- 工作产出/效果与**参与人数**挂钩，但遵循边际递减（与 v2 科研的人力上限一致）。
- 多数工作会加速心情下降，只有「休息」能恢复心情——形成必须权衡的张力：派太多人干活会集体崩溃，必须留人休息。

### 工作清单（第一版：4 项可分配工作）

每项工作都精确对应一个已有资源或系统，不新增冗余机制。**每个角色每天从这 4 项中选 1 项：**

| 工作 | 效果 | 代价 | 关联系统 |
|------|------|------|----------|
| 🔬 科研 | 推进所在实验室的课题进度 | 心情↓更快、耗电 | v2 科研系统 |
| 😴 休息 | **回复心情与健康**（唯一恢复手段） | 不产出 | 心情/健康系统 |
| 🔧 修复基地 | 消耗建筑材料，恢复 `hull_integrity` | 耗材料、心情↓ | 突发事件/结构系统 |
| ⛏️ 探索采集 | 获取净水（月壤冰）、建筑材料（月壤） | 耗电、出舱**辐射暴露**扣健康/心情 | 净水、材料、辐射 |

### 耕作为自动机制（不占工作位）

**耕作不作为可分配工作**，而是**自动进行**：温室舱/种植舱只要存在就自动照料作物，其产出与消耗按舱室数量自动结算（沿用现有农业逻辑，无需派人）。这样角色的 4 项工作保持精简，农业则始终在后台运转。

> 后续可加（先不做）：**值班运维**（降低突发事件概率/缩短持续）、**医疗照护**（加速低心情或受辐射成员恢复）。等基础跑顺再扩。

### 关键平衡点

- **休息是刚需**：所有产出型工作都扣心情，必须有人轮休，否则心情单向下滑直至兵变；休息同时恢复心情与健康。
- **出舱有真实风险**：探索采集的辐射暴露让它不能无脑刷，月壤护甲（已有 `Regolith_Shield_m2`）越厚则出舱扣健康/心情越少——让护甲投入再次增值。
- **健康与心情双轨**：健康反映身体状态（辐射、缺氧、过劳），心情反映心理状态，两者分别恢复、分别可致崩溃。

### 产出公式（建议形式）

```
某工作当日效果 = 基础效果 × 有效人力系数
有效人力系数 = 边际递减(参与人数)   例如 1人=1.0, 2人=1.7, 3人=2.2 ...
心情变化：产出型工作每人额外 −X 心情/天；休息每人 +Y 心情/天
```

## 二、界面结构重构：参数锁定

### 现状问题

当前侧边栏把**初始配置**（初始氧气瓶等）和**游戏中操作**（建舱、播种、微藻调控、推演）混在一起，且舱室数量、乘组规模等可在游戏中随时改动——这让「经营决策」失去了重量，玩家可随时反悔。

### 新结构：三阶段分离

**阶段 1 · 初始化（游戏开始前，仅一次）**
将以下设定移入一个**开局配置界面**，确认后锁定，游戏中不可改：
- 初始高压氧气瓶数量
- 初始乘组规模与人员
- 初始舱室配置（居住/种植/堆肥的起始数量）
- 其他开局战略物资

实现方式建议：用 `st.session_state.game_started` 标志位。未开始时只显示配置界面 + 「🚀 确认配置，开始任务」按钮；点击后置 `game_started=True` 并锁定这些参数。

**阶段 2 · 每日运营（游戏进行中，可操作）**
游戏中**仍可操作**的是「运营决策」，而非「初始设定」：
- 角色工作分配（本版新增，每日重置）
- 播种指令、微藻光水调控、应急干预（焚化炉/排洪）
- 建造/拆除舱室（消耗资源，作为游戏内行为而非自由调参）
- 推演时间

**阶段 3 · 锁定参数（游戏中只读展示）**
初始化设定的参数在游戏中以**只读**形式展示（如侧边栏顶部显示「乘组 4 人 ｜ 着陆携带 10 瓶氧」），让玩家清楚自己的开局约束，但不能改。

> 关键原则：**初始设定一旦锁定不可改 = 决策有后果**；游戏中能动的只有"今天做什么"，不能动"我当初带了多少"。

## 三、新增角色界面（Crew Panel）

### 展示内容

一个独立的角色面板（可放主区顶部或单独 expander），逐人显示：

| 字段 | 说明 |
|------|------|
| 姓名 | 已有 `crew_list` 的 `name` |
| 心情 | 已有 `mood`，按三阶段着色（>70 绿 / 40–70 黄 / <40 红）|
| 健康 | 新增字段（辐射暴露、缺氧等会扣健康，与心情区分）|
| 今日工作 | 下拉/按钮分配本日工作 |
| 状态标记 | 如「出舱中」「科研中」「休息中」「⚠️低心情」|

### 分配交互

- 每个角色一个工作选择控件（`selectbox` 或按钮组）。
- 分配后，下一次「闭环演进」时按各人工作结算产出与心情。
- 每天（每次推演步）工作分配**可重新调整**——工作是"今天的安排"，不是永久职业。

### 与推演的衔接

`step_system` 需要接收每个角色的工作分配（如一个 `{name: job}` 字典），在每日循环里：
1. 按工作类型分别结算产出（科研进度、采集资源、修复壳体）；
2. 按工作类型施加差异化的心情/健康变化；
3. 休息的人恢复心情与健康，出舱采集的人承受辐射（扣健康，护甲可减免）；
4. 耕作/农业自动结算，不依赖工作分配。

## 系统逻辑需要修改的地方（实装清单）

1. **新增 `game_started` 状态机**：分离初始化与运营两个阶段，初始化界面 + 锁定逻辑。
2. **crew 数据结构扩展**：每个成员增加 `job`（今日工作）和 `health`（健康值，0–100）字段。
3. **角色界面**：新增逐人面板，含工作分配控件（4 选 1）。
4. **`step_system` 改造**：接收工作分配，按工作分别结算产出与心情/健康；农业自动结算。
5. **侧边栏拆分**：初始设定移入开局界面并锁定；运营操作（建造/拆除/播种/调控/分配工作）保留；锁定参数改为只读展示。
6. **心情/健康公式整合**：把各工作的心情、健康增减并入每日结算；新增健康归零的死亡判定。

## 已确定的设计决策

1. **工作清单**：第一版做「科研 / 休息 / 修复 / 采集」**4 项**。耕作改为**自动机制**，不占工作位。值班、医疗暂缓。
2. **健康值**：**新增独立「健康」字段**，与心情区分。辐射、缺氧、过劳扣健康；休息恢复健康。心情管心理、健康管身体，两者分别可致崩溃。
3. **建造 / 拆除**：保留为**游戏中的运营行为**（消耗资源主动建/拆），不锁进初始化。
4. **新成员与补给**：作为**特殊好事件（正面突发事件）**加入——补给事件可补充物资，新成员事件可增加乘组规模。这意味着乘组规模初始锁定，但可在游戏中通过好事件**增加**（而非玩家自由调参）。

## 仍需你确认

1. **锁定范围**：初始化锁定哪些参数？建议锁定「初始氧气瓶 / 初始乘组规模 / 初始舱室配置」三项。
2. **健康值的死亡判定**：健康归零是否触发死亡（如同心情归零的兵变）？建议是。
3. **依赖顺序**：本版工作分配依赖 v2（科研/电力）、v3（温室/净水）。是否先把前序系统实装，再做工作分配？还是工作分配先用占位接口（科研/采集先给简化产出）？

—— 这几点确认后即可动手改代码。建议的实装顺序：先重构界面（初始化锁定 + 状态机），再加健康值与角色面板，最后接入 4 项工作的结算逻辑。
